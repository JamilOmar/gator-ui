About

This is a tool to manage your development team. Nothing more, nothing less.
This is a tool for senior managers, who would like to keep their hand on the pulse. It is not about the micromanagement but getting the heat map of the team. Knowing what is happening on the shop floor.
The senior manager gets the high-level view, and if needed they can quickly drill down. First level manager, Leads can have a more zoomed-in view.

This tool covers the whole spectrum from first-level managers to senior Director, VP and CTO need.

Second, a big problem solved by this tool is - simplifying status reporting.

It is easy to write, add your git PR, Jira Tickets with one click. Every report can be reviewed by managers, rate the work with three checkboxes â€“ Achieved, Exceeded and Need Improvement.
All the reports are archived and one-click access to all the history within tools, help tremendously with the yearly, half-yearly review process. The review process is no more scary time, where managers spend hours preparing for discussion. All the information they need is a click away. Because the manager is forced to write their comments and rating every month. Final reviews will have no more surprises.
We are not reinventing the wheel, we are not trying to replace existing awesome tools. We are aggregating information from different tools, for a one glance quick status.

Use Dev Star one screen, a quick glance to know:

  • How many pull request open/closed today, last seven days and last 30 days.
  • Monitor multiple organizations Pull Request/Jira Tickets from one screen.
  • Monitor multiple developers from one screen.
  • Drill down in a Jira tickets for the developer.
  • Zero down to one repository to see all the pull request.
  • One click to drill down view of Jira and Git.
  • Simplyfy Status reporting.
  • Easy access to all the historical status reports.
  • Watch today's pull request details scrolling on top of the screen
